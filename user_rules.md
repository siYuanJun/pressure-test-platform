# 开发执行规则（请严格遵守）
## 前置要求【核心】
1. 开始本次开发任务前，必须先完整读取项目根目录的以下3个文件，掌握项目进度、结构和历史变更：
   - STATUS.md：项目进度看板（完成度、待办、阻塞项）
   - CHANGELOG.md：功能变更日志（新增/修复/调整记录）
   - PROJECT_MAP.md：项目结构心智图（文件路径+功能说明）
2. 读取后需确认：本次开发的文件路径、功能是否与已有内容冲突，避免重复开发

## 基础规范
1. 全程使用中文沟通与注释，代码注释也为中文
2. 开发环境适配Mac OS（脚本/命令需兼容Mac，避免Linux独有指令）
3. 所有代码必须添加函数级注释（Python遵循PEP8，JS/TS遵循ESLint规范）
4. 若为Node.js项目，严格适配.eslintrc.js、prettier.config.js编码规范（无配置文件时按主流React/Next.js规范）
5. 若为 python3 项目必须在 source venv/bin/activate 下执行 python3 命令 pip install 安装依赖

## 项目管理要求
完成本次开发任务后，必须按以下规则更新项目核心文件（仅补充内容，不修改原有结构）：
### 1. STATUS.md
- 更新「整体进度」模块完成度（百分比）
- 「已完成功能」添加本次开发成果（格式：- [x] 功能描述）
- 「待完成功能」补充后续依赖项（如有）
- 「阻塞项」记录开发中遇到的问题（如有）

### 2. CHANGELOG.md
- **版本号递增规则**：每次AI完成一项开发任务（即对话结束时），必须以版本号递增的形式更新CHANGELOG.md
  - 版本号格式：遵循语义化版本控制（Major.Minor.Patch），如0.1.0 → 0.2.0
  - 递增时机：每次完整任务完成后立即执行版本升级
  - 不需要日期
- **内容格式要求**：
  - Added：本次新增的功能/文件（格式：- 目录/文件：功能说明）
  - Fixed：本次修复的问题（如有）
  - Changed：本次调整的逻辑/结构（如有）

### 3. PROJECT_MAP.md
- 新增/修改文件时，按照项目结构树的格式补充，即「目录层级+文件+说明」的树状结构
- 仅记录文件的功能用途，不编写具体代码实现
- 关键接口或函数仅标注核心的入参和出参信息（无需完整参数列表）

## 交付要求
1. 代码可直接在Mac环境运行，提供清晰的启动/执行命令
2. 依赖清单（requirements.txt/package.json）明确版本号
3. 每个模块目录下补充README.md（说明文件结构、使用方法）
